Feather Framework

Feather is a lightweight modular architecture tailored for performance-critical games. By enforcing clean initialization pipelines and rigid type guards, it minimizes framework footprint while keeping client-server communication fast and structural tracking instantaneous.

Strict Type Safety Execution: Feather operates flawlessly inside strict environments (--!strict). Declaring explicitly mapped custom structural shapes prevents interface typing bugs before your framework compiler initializes modules.

Building Controllers

Controllers serve as your execution controllers across game operations. They are declared cleanly and automatically returned dynamically via registration framework wrappers.

--!strict
local ReplicatedStorage = game:GetService("ReplicatedStorage")
local Feather = require(ReplicatedStorage.Feather)

export type TestControllerType = {
    featherInit: (self: any) -> (),
    featherStart: (self: any) -> (),
}

local TestController = {} :: TestControllerType

function TestController:featherInit()
    print("⚙️ [TestController]: Running pre-init sequences...")
end

function TestController:featherStart()
    print("🚀 [TestController]: Initial startup confirmed!")
end

return Feather.CreateController(TestController)

Framework Lifecycle Hooks

Feather looks for specific camelCase layout method pointers inside your modules to cleanly step through startup phases sequentially:

  • featherInit() — Fires instantly when the controller is processed. Ideal for micro-setting allocations, baseline memory configs, and utility setups.
  • featherStart() — Fires once every registration block and networking loop has resolved. Safe for multi-threaded setups, server queries, and active connection patterns.
