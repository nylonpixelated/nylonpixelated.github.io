Feather Framework

Feather is a lightweight, ultra-high performance modular architecture tailored for production-scale Roblox game environments. By enforcing clean initialization pipelines and strict type safety layouts, it eliminates cyclic dependencies and provides instant response times under heavy simulation load.

Strict Type Safety Execution: Feather operates flawlessly inside strict environments (--!strict). Declaring explicitly mapped custom structural shapes prevents runtime mapping interface bugs before your framework compiler initializes modules.

Server Services

Services manage server-side singleton modules and runtime tasks. Use explicit type declarations to handle database updates, matchmaking networks, and system tracking.

--!strict
local ReplicatedStorage = game:GetService("ReplicatedStorage")
local Feather = require(ReplicatedStorage.Feather)

export type DataServiceType = {
    Client: {},
    featherInit: (self: any) -> (),
    featherStart: (self: any) -> (),
}

local DataService = { Client = {} } :: DataServiceType

function DataService:featherInit()
    print("🔒 [DataService]: Initializing DataService...")
end

function DataService:featherStart()
    print("🚀 [DataService]: Listening for data updates!")
end

return Feather.CreateService(DataService)

Client Controllers

Controllers isolate client logic execution boundaries. They compile instantly and return via framework wrappers dynamically.

--!strict
local ReplicatedStorage = game:GetService("ReplicatedStorage")
local Feather = require(ReplicatedStorage.Feather)

export type TestControllerType = {
    featherInit: (self: any) -> (),
    featherStart: (self: any) -> (),
}

local TestController = {} :: TestControllerType

function TestController:featherInit()
    print("⚙️ [TestController]: Allocating micro-setting buffers...")
end

function TestController:featherStart()
    print("🚀 [TestController]: Main client execution tracking active!")
end

return Feather.CreateController(TestController)

Framework Lifecycle Hooks

Feather drives step-by-step initialization loops sequentially using internal camelCase hooks:

  • featherInit() — Fires synchronously when the module is processed. Safe for configuration adjustments, memory allocations, and variable caches.
  • featherStart() — Fires once every module registration block has cleared safely. Optimized for handling active async processes, server remote mapping, and thread routines.

FeatherSpring

A completely customized, high-precision linear vector spring solver module. Built explicitly to drive physical interpolation pipelines, UI/UX liquid motions, and reactive screen shaking effects instantly.

local FeatherSpring = require(ReplicatedStorage.Packages.FeatherSpring)

local elasticSpring = FeatherSpring.new(1, 40, 4, 0) -- Target, Speed, Damper, Velocity
elasticSpring:Update(deltaTime)

FeatherCleanup

A fast tracker object engineered to clean up complex connections and track items to prevent memory leaks across temporary data structures.

local FeatherCleanup = require(ReplicatedStorage.Packages.FeatherCleanup)
local Janitor = FeatherCleanup.new()

Janitor:Add(workspace.Part, "Destroy")
Janitor:Clean() -- Safely dumps garbage collection elements instantly

FeatherSignal

An execution event architecture that replaces slow bindable objects. It handles thread management inside multi-threaded micro-tasks perfectly.

local FeatherSignal = require(ReplicatedStorage.Packages.FeatherSignal)
local onAbilityUsed = FeatherSignal.new()

onAbilityUsed:Connect(function(abilityName)
    print("Casted skill:", abilityName)
end)

FeatherNet

An optimized communication layer routing structural commands across the network border cleanly. Includes micro-serialization handling to keep bandwidth tight.

local FeatherNet = require(ReplicatedStorage.Packages.FeatherNet)

-- Server Side Event Setup
local ReliableRemote = FeatherNet.CreateEvent("ReliableRemote")
ReliableRemote:FireAllClients("MatchStart")
